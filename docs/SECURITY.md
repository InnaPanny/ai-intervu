# 安全与隐私开发基线

## 敏感信息范围

以下数据必须视为敏感信息：

- API Key、数据库密码、腾讯云密钥和短信服务密钥。
- 手机号、验证码和登录会话。
- 简历、工作经历、面试答案和现实面试复盘。
- 用户上传的原始文件及其提取文本。

## 密钥管理

- 密钥只允许写入本地 `.env.local` 或正式环境的密钥管理服务。
- 不得将真实密钥写入源码、文档、测试、日志、截图、聊天记录或 Git。
- 浏览器端不得读取服务端密钥；禁止使用 `NEXT_PUBLIC_` 前缀保存密钥。
- 密钥一旦出现在聊天、日志或代码中，应立即撤销并重新生成。
- `.env.example` 只保留空值和非敏感默认值。

## 简历文件

- 首版仅支持 PDF 和 DOCX，最大 5MB。
- 同时校验文件扩展名和文件签名，不只信任浏览器提供的 MIME 类型。
- 当前 MVP 只在服务端内存中提取文本，不保存上传的原文件。
- 接口和日志不得记录文件名、原始文件、提取文本或解析器详细错误。
- 解析失败时允许用户重新上传或直接粘贴文本。
- 正式接入 COS 后，使用私有存储桶、短时签名 URL、服务端授权和生命周期策略。

## AI 与日志

- AI 只在服务端调用，手机号和验证码不得发送给模型。
- 向 AI 发送完成任务所需的最少信息。
- 单题参考应答只发送当前题目、题型、用户本次回答、目标岗位与行业及信息缺口，不发送完整简历。
- 方法、认知和通用题可由模型补充专业框架，但不得声称为用户个人经历或成果。
- 经历和个人定位题需拒绝模型新增的个人数字和模型自行标记为无法确认的事实。
- 已配置远程 AI 时，调用失败或输出不满足质量与事实约束必须明确报错，不得伪装为 AI 生成结果。
- 外部服务错误对用户返回通用信息，不透出请求内容、密钥、上游响应体或内部堆栈。
- 禁止使用 `console.log` 输出敏感请求、用户资料或模型输入输出。

## 上线前必做

- 接入正式认证与接口限流。
- 对上传文件执行恶意文件扫描和隔离。
- 使用中国大陆地域数据库和对象存储。
- 配置密钥轮换、审计日志、备份与恢复。
- 由专业人员复核隐私政策、用户协议和个人信息处理流程。
