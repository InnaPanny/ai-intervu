# AI 面试教练 MVP 研发实施计划

## 1. 实施策略

优先完成响应式网页 MVP。采用纵向切片方式开发，每个阶段都交付一条可运行、可验收的用户流程，避免先堆积无法使用的孤立模块。

## 2. 已确认事项与待确认事项

已确认：

- AI 模型服务商使用 DeepSeek，且只能由服务端调用。
- 第一版支持 PDF、DOCX 简历上传，同时保留粘贴简历文本。
- 内部开发阶段使用固定测试验证码。
- MVP 暂不开发管理后台。

仍待确认：

- 正式产品名称和网页标题。

## 3. 推荐技术方案

以下为当前生效方案。中国市场架构决策详见 `docs/ARCHITECTURE_DECISION_CHINA.md`。

- 前端与服务端：Next.js + TypeScript。
- UI：Tailwind CSS + shadcn/ui。
- 数据库：腾讯云 TencentDB for PostgreSQL。
- 文件存储：腾讯云 COS。
- 手机验证码：腾讯云 SMS。
- AI：DeepSeek 官方 API，通过服务端统一调用并使用结构化 JSON 输出。
- 语义去重：先使用模型生成题目“考察意图标签”与相似度判断；数据量增加后再引入向量检索。
- 部署：Vercel。

选择理由：

- 一套代码同时实现响应式网页和服务端接口。
- 适合快速完成 MVP，并保留后续扩展微信小程序所需的 API 边界。
- 中国大陆地域服务更符合目标市场的访问稳定性和个人信息处理要求。

## 4. 分阶段实施

### 阶段 0：产品与工程基线

交付内容：

- 冻结 MVP PRD。
- 建立页面地图和核心流程低保真原型。
- 确定技术方案。
- 初始化项目、数据库迁移和环境变量模板。

验收标准：

- 产品关键页面和状态流转无阻塞性歧义。
- 本地可以启动响应式网页。
- 数据库可以创建测试用户和求职目标。

### 阶段 1：账户、求职目标与页面骨架

交付内容：

- 测试手机号登录。
- 五个一级导航。
- 求职目标创建、切换与管理。
- 首次使用核心功能时的求职目标弹窗。
- 首页基础状态。

验收标准：

- 新用户可以登录并创建求职目标。
- 页面在桌面与手机宽度下均可使用。
- 切换求职目标后，各目标的数据上下文独立。

### 阶段 2：动态题目生成与个人题目库

交付内容：

- AI 动态生成最多 15 道题及最多 7 道核心题。
- 推荐原因与考察重点。
- 待练题列表和已刷题列表。
- “不再显示”后台状态与后续去重。
- 基于历史题目和考察意图的语义去重。

验收标准：

- 重复刷新不会重复生成同一考察意图的问题。
- 原有未作答题不会因生成新题而消失。
- 不再显示的题目不会在同一求职目标下重新出现。

### 阶段 3：单题练习与个人答案

交付内容：

- 开始作答、梳理回答思路、查看回答结构三个入口。
- 信息缺口识别与展示，MVP 不进行追问。
- 维度评级、具体建议和信息缺口。
- 回答结构、关键词提纲和完整参考答案。
- 用户编辑、确认并保存个人最终答案。
- 有帮助或没帮助反馈。

验收标准：

- AI 不会在用户未提供时自行补全经历和数据。
- 用户中途结束后，已完成内容仍被保存。
- 未经用户确认的 AI 参考答案不会成为个人最终答案。

### 阶段 4：重点练习与今日复习

交付内容：

- 用户手动加入重点练习。
- 三种记忆训练方式。
- 掌握程度判断与提示使用记录。
- 当日复习、次日复习和已完成强化规则。
- 首页今日复习入口。

验收标准：

- 需要加强的题目能在结束连续练习后进入当日复习。
- 连续两次间隔复习表现优秀后，题目自动完成强化。
- 历史练习记录不会因状态变化丢失。

### 阶段 5：现实面试复盘

交付内容：

- 按面试日期倒序排列的复盘列表。
- 新建、保存草稿、继续补充与完成复盘。
- 逐道手动添加现实面试问题和实际回答。
- 主动生成 AI 参考应答并确认个人最终答案。
- 用户手动分类到待练题、重点练习、仅保存记录或不再显示。

验收标准：

- 只填写岗位和日期即可保存草稿。
- 完成复盘时至少包含一道问题。
- 现实面试问题会参与后续题目语义去重。

### 阶段 6：整体验收与上线准备

交付内容：

- 主流程端到端测试。
- 移动端响应式检查。
- AI 输出异常与重试处理。
- 隐私政策和用户协议占位页面。
- 部署测试环境。

验收标准：

- PRD 中的 MVP 验收主流程可以完整跑通。
- 关键页面不存在阻塞操作的严重错误。
- AI 调用失败时，用户内容不会丢失。

## 5. 核心数据对象

开发时至少需要以下数据对象：

- 用户。
- 求职目标。
- 用户资料与文件。
- 题目。
- 题目生成批次。
- 题目与求职目标的状态。
- 练习会话。
- 用户回答。
- AI 评价与参考答案。
- 个人最终答案。
- 重点练习计划与复习记录。
- 现实面试复盘。
- 现实面试问答。
- 用户反馈。

详细字段和关系在阶段 0 技术设计中补充。

## 6. 主要风险

- AI 可能虚构信息，需要通过提示词、结构化输出和用户确认共同约束。
- 语义去重并非绝对准确，需要保留用户“不再显示”和反馈机制。
- 动态题目质量依赖用户提供的信息，需要在资料不足时明确提示。
- 手机验证码会产生费用和合规要求，开发阶段建议使用测试验证码。
- 简历文件包含高敏感个人信息，必须限制为 PDF、DOCX，执行大小和文件签名校验，并在正式上线前接入恶意文件扫描。
