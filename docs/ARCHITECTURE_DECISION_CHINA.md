# 中国市场技术架构决策

决策日期：2026-06-14

## 结论

正式面向中国大陆用户上线时：

- 不采用 Supabase 托管云作为主后端。
- 数据库采用腾讯云 TencentDB for PostgreSQL。
- 简历等文件采用腾讯云 COS。
- 手机验证码采用腾讯云 SMS。
- AI 模型采用 DeepSeek 官方 API。
- 当前 Next.js 服务端继续作为统一业务 API 层，避免浏览器直接访问数据库和模型。

Supabase 迁移文件暂时保留，仅用于参考和快速原型，不作为中国市场正式生产方案。

## Supabase 评估

Supabase 提供数据库、认证、文件存储和权限控制，开发体验很好，适合快速原型。但当前托管区域没有中国大陆，亚太区域主要为新加坡、东京和首尔。

本产品会处理手机号、简历、工作经历、面试答案和现实面试复盘。这些内容涉及个人信息。使用境外托管服务会增加：

- 中国大陆访问速度和稳定性风险。
- 个人信息跨境告知、单独同意和影响评估等合规工作。
- 国内手机号验证码服务适配成本。

因此，Supabase 托管云适合内部原型，不适合作为本产品正式生产后端的默认选择。

## 腾讯云方案

- TencentDB for PostgreSQL：保存用户、求职目标、题目、答案和复盘。
- COS：保存用户主动上传的简历文件。
- SMS：发送中国大陆手机号验证码。
- Next.js 服务端：统一处理登录、数据访问、AI 调用和权限校验。

选择腾讯云的原因：

- 支持中国大陆地域，访问和数据存储更适合目标市场。
- PostgreSQL 与当前数据模型兼容。
- 腾讯云短信适用于中国大陆手机号验证码。
- 后续开发微信小程序时协作更顺畅。

## DeepSeek V4 模型策略

DeepSeek 官方 API 当前提供 `deepseek-v4-flash` 和 `deepseek-v4-pro`，两者均支持 JSON 输出。

| 工作 | 模型 |
|---|---|
| 批量生成面试题、语义去重、题目分类、简单初评 | V4 Flash |
| 深度回答辅导、最终参考答案 | V4 Pro |

这样可以兼顾成本、速度和回答质量。技术上继续保留统一 AI 接口，未来替换模型或增加备用供应商时，不需要重写产品页面。

## 合规原则

- 简历和面试记录默认存储在中国大陆地域。
- 调用 AI 前，仅传递完成任务所需的最少信息。
- 不将手机号发送给 AI 模型。
- 用户应知晓其内容会被发送给 AI 服务进行处理。
- AI 不得虚构用户经历、职责、成果或数据。
- 正式上线前，由专业人员复核隐私政策、用户协议和个人信息处理流程。
